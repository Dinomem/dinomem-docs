# DinoMem

**The memory layer for AI agents.** Postgres-native, TypeScript- and Python-first, framework-agnostic. Drop it into any agent stack (LangChain, LangGraph, CrewAI, Mastra, Claude SDK, custom) in three lines.

```ts copy
import { MemoryStore } from '@dinomem/sdk'

const mem = new MemoryStore({ apiKey: process.env.DINOMEM_API_KEY! })

await mem.write({ content: 'Alice prefers email over phone', agentId: 'support-bot' })
const hits = await mem.search({ query: 'how should I reach Alice?', agentId: 'support-bot' })
```

## What you get

- **Write + search** with embeddings, BM25 keyword search, and entity-graph traversal — all merged with Reciprocal Rank Fusion. Optional [Gemini re-ranking](/features/reranking) for high-precision queries.
- **Multi-agent coordination.** [Scopes](/concepts) (private / team / global) + roles (planner / executor / observer) + [access grants](/features/access-grants) give you fine-grained visibility control out of the box.
- **CRDT conflict resolution.** Vector clocks per agent, [configurable policies](/features/conflict-policies) (`planner_wins`, `timestamp_wins`, `human_in_loop`), and [supersession lineage](/features/history) so you can audit who replaced what.
- **TTL + retention.** [Memories expire](/features/ttl) automatically. [Retention policies](/features/retention) prune old data on a schedule.
- **Webhooks.** Subscribe to `memory.written`, `memory.expired`, `memory.conflict_detected`, etc. — HMAC-signed payloads.
- **BYOK encryption** for compliance-sensitive deployments.

## Install

```bash copy
# TypeScript
npm install @dinomem/sdk

# Python
pip install dinomem-py
```
