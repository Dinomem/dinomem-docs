# Getting started

This walks you from "never heard of DinoMem" to "memory working in an agent" in about three minutes.

## 1. Sign up and get an API key

1. Open the dashboard: [dinomem-dashboard.vercel.app](https://dinomem-dashboard.vercel.app)
2. Click **"No account? Sign up"** — enter your email, a password, and (optionally) an organization name. An org is created for you automatically.
3. Confirm the email Supabase sends you (or sign in directly if email confirmation is off for the project).
4. You'll land on the **API Keys** tab. Click **Create key**, give it a name, copy the `sk-...` shown once — the dashboard won't show it again.

Keys are scoped to your org. To bring in a teammate, go to the **Members** tab → invite by email → share the generated link. They sign up via that link and land in your org instead of getting a new one.

## 2. Install the SDK

The fastest path is the init CLI — it detects your project, installs the SDK, drops an env placeholder, and writes a demo script:

```bash copy
npx @dinomem/cli init
```

## 3. Write your first memory

<Tabs items={['TypeScript', 'Python']}>
<Tabs.Tab>
```ts copy
import { MemoryStore } from '@dinomem/sdk'

const mem = new MemoryStore({ apiKey: process.env.DINOMEM_API_KEY! })

const { writeId } = await mem.write({
  content: 'Customer prefers email over phone. Escalation threshold is $500.',
  agentId: 'support-bot',
  scope:   'team',          // private | team | global — see Concepts
  role:    'executor',      // optional — used by conflict policies
  workflowId: 'wf-123',     // optional — groups related memories
})

console.log({ writeId })
```
</Tabs.Tab>
<Tabs.Tab>
```python copy
import os
from dinomem_py import MemoryStore

mem = MemoryStore(api_key=os.environ["DINOMEM_API_KEY"])

result = mem.write(
    content="Customer prefers email over phone. Escalation threshold is $500.",
    agent_id="support-bot",
    scope="team",
    role="executor",
    workflow_id="wf-123",
)
print(result.write_id)
```
</Tabs.Tab>
</Tabs>

The call returns as soon as the memory is written and indexed. Entity extraction (for graph search) happens asynchronously in the background.

## 4. Search

<Tabs items={['TypeScript', 'Python']}>
<Tabs.Tab>
```ts copy
const hits = await mem.search({
  query:   'how do we handle customer escalations?',
  agentId: 'support-bot',
  topK:    5,
})

for (const hit of hits) {
  console.log(hit.score, hit.content)
}
```
</Tabs.Tab>
<Tabs.Tab>
```python copy
hits = mem.search(
    query="how do we handle customer escalations?",
    agent_id="support-bot",
    top_k=5,
)
for hit in hits:
    print(hit.score, hit.content)
```
</Tabs.Tab>
</Tabs>

Under the hood, DinoMem runs three retrieval strategies in parallel — semantic similarity (pgvector), keyword (BM25), and entity-graph traversal — and merges them with Reciprocal Rank Fusion. The returned `score` is the fused score.

For higher precision (at the cost of ~500–1500ms of latency), pass `rerank: true` and DinoMem will re-score the top candidates with Gemini. See [Re-ranking](/features/reranking).

## REST equivalent

Every SDK call is a thin wrapper over the REST API. The above two calls in `curl`:

```bash copy
curl -X POST https://lwbwcuuzoituanwhekyo.supabase.co/functions/v1/api/v1/memory/write \
  -H "Authorization: Bearer $DINOMEM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"content":"Customer prefers email","agentId":"support-bot","scope":"team"}'

curl -X POST https://lwbwcuuzoituanwhekyo.supabase.co/functions/v1/api/v1/memory/search \
  -H "Authorization: Bearer $DINOMEM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"query":"customer escalations","agentId":"support-bot","topK":5}'
```
